# gcc-cross-zig

Shared Zig build modules for compiling GNU Binutils 2.42 and GCC 14.2.0 cross-compilers from source.

This repo contains only build logic. It does not produce binaries on its own. Target-specific repos consume these modules:

- [gcc-rl78-zig](https://github.com/Bud-ro/gcc-rl78-zig) -- Renesas RL78 (16-bit)
- [gcc-rx-zig](https://github.com/Bud-ro/gcc-rx-zig) -- Renesas RX (32-bit CISC)
- [gcc-v850-zig](https://github.com/Bud-ro/gcc-v850-zig) -- Renesas V850 (32-bit, automotive)
- [gcc-sh-zig](https://github.com/Bud-ro/gcc-sh-zig) -- Renesas SuperH (32-bit RISC)
- [gcc-h8300-zig](https://github.com/Bud-ro/gcc-h8300-zig) -- Renesas H8/300 (16/32-bit)

## How it works

Each target repo (e.g. gcc-rl78-zig) depends on this package and provides:

- **`generated/<target>/`** -- Pre-generated GCC files (insn-\*.cc/h, options.h/cc, gtype-desc.\*, match-\*.cc) produced by GCC's gen\* host tools and AWK scripts. Deterministic for a given GCC version + target.
- **`config/<target>/`** -- Platform config headers (auto-host.h, tm.h) replacing what `configure` detects.
- **`vendor/ld/`** -- Vendored linker emulation file (generated by shell scripts from ld templates).
- **`include/`** -- Target-specific header redirects for gas/ld (targ-cpu.h, obj-format.h, etc).

This repo provides the parameterized build functions that compile everything:

| Module | What it builds |
|--------|---------------|
| `binutils_libs.zig` | libiberty, libsframe, libbfd, libopcodes |
| `binutils_gas.zig` | Cross-assembler (e.g. rl78-elf-as) |
| `binutils_ld.zig` | Cross-linker (e.g. rl78-elf-ld) |
| `binutils_tools.zig` | objcopy, objdump, readelf, ar, nm, strip, size, ranlib, strings, addr2line, c++filt, elfedit |
| `gcc_cc1.zig` | cc1 (compiler proper), gcc driver, libcpp, libdecnumber, libcody |
| `cross_config.zig` | Shared types (CrossConfig, Libs) |
| `find_replace.zig` | Build-time text replacement utility |

Source tarballs (binutils, GCC) are declared by the consumer and downloaded automatically by Zig on first build from [ftpmirror.gnu.org](https://ftpmirror.gnu.org).

## Adding a new target

1. Create a new repo (e.g. `gcc-sh-zig`)
2. Add `gcc-cross-zig` as a dependency in `build.zig.zon`
3. Add binutils and GCC source tarballs as URL dependencies
4. Generate the target-specific files from a one-time `configure && make` build
5. Write a `build.zig` that imports `gcc_cross` and calls `buildToolchain()` with your `CrossConfig`
